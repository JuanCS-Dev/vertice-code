# GDPR Compliance Implementation - Vertice-Chat SaaS

## Data Mapping & Inventory

### Personal Data Collected

#### 1. Account Data (Article 6(1)(b) - Contract)
- **Email address**: Firebase Auth primary identifier
- **Name**: User profile information
- **Profile picture**: Optional user upload
- **Account creation date**: System generated
- **Last login**: System tracked

#### 2. Usage Data (Article 6(1)(f) - Legitimate Interest)
- **AI chat interactions**: Messages, responses, timestamps
- **Feature usage**: Which tools accessed, frequency
- **Performance metrics**: Response times, error rates
- **Device information**: Browser, OS (anonymized)
- **Geolocation**: Country-level only (anonymized)

#### 3. Billing Data (Article 6(1)(b) - Contract)
- **Payment information**: Processed by Stripe (we don't store)
- **Subscription details**: Plan type, billing cycle
- **Usage billing**: Token consumption, feature usage
- **Invoice history**: Generated by Stripe

#### 4. Technical Data (Article 6(1)(f) - Legitimate Interest)
- **IP addresses**: For security (7-day retention)
- **Session tokens**: Authentication (24-hour expiry)
- **Error logs**: For debugging (30-day retention)
- **Audit logs**: Security events (7-year retention)

### Data Processors

| Processor | Purpose | Location | Data Types | Retention |
|-----------|---------|----------|------------|-----------|
| Firebase Auth | Authentication | Global | Email, name | Account active |
| Firestore | Database | Multi-region | All user data | Account active |
| Vertex AI | AI processing | US | Chat messages | 30 days |
| Stripe | Payments | US | Billing data | 7 years |
| Cloud Logging | Monitoring | Global | Logs | 30 days |

### Data Retention Schedule

| Data Type | Retention Period | Legal Basis | Disposal Method |
|-----------|-----------------|-------------|-----------------|
| Account data | Account active + 3 years | Contract | Database deletion |
| Chat history | 30 days (configurable) | Legitimate interest | Auto-deletion |
| Billing data | 7 years | Legal obligation | Archive |
| Audit logs | 7 years | Legal obligation | Secure archive |
| Error logs | 30 days | Legitimate interest | Auto-deletion |

### Risk Assessment (DPIA)

#### High Risk Activities:
1. **AI Data Processing**: Automated decision-making with personal data
2. **Large-scale profiling**: Usage patterns analysis
3. **Cross-border transfers**: Data stored globally

#### Mitigation Measures:
- Data minimization (collect only necessary data)
- Purpose limitation (clear data usage policies)
- Storage limitation (automated deletion)
- Security measures (encryption, access controls)
- User rights (easy access to data controls)
